> Packages in this directory are consumed by both `apps/api` and `apps/frontend`.
> They must never contain app-specific logic - only shared contracts, types, and schemas.

---

## Package Index

| Package | Purpose |
|---|---|
| `packages/swag/` | Orval-generated TypeScript API client and types from `swagger-spec.json` |
| `packages/schemas/` | Shared Zod schemas and the `ErrorCode` enum used by both frontend and backend |
| `packages/tsconfig/` | Shared TypeScript base configs extended by all apps |

---

## Rules That Apply to All Packages

- Packages export **types, schemas, and generated code only** - never business logic, never service calls.
- No package may import from `apps/` - dependency flows one way: `apps/ -> packages/`.
- Each package has its own `package.json` and `tsconfig.json` extending `packages/tsconfig/base.json`.
- All packages are built before apps - define this dependency order in `turbo.json`.
- Exported shared types, schemas, DTO-aligned fields, and client payload shapes stay `camelCase` at the field level; exported type, enum, and schema names use `PascalCase`.

## `packages/swag/`

Contains the TypeScript API client generated by **Orval** from `swagger-spec.json`.

**Do not hand-edit any file in `src/`.** Everything here is generated - manual edits will be overwritten.

**Regeneration:**

```bash
pnpm codegen   # runs orval, reads swagger-spec.json, writes packages/swag/src/api.ts
```

Run this after every backend contract change (new endpoint, changed DTO, new response field).

**When to commit:** `packages/swag/src/api.ts` must be committed alongside the backend change that triggered it. A PR that changes a controller or DTO without updating the generated client will cause type errors on the frontend.

**Orval config lives in `packages/swag/orval.config.ts`** - update it if the backend base URL, auth header strategy, or output format changes.

## `packages/schemas/`

Contains Zod schemas and the `ErrorCode` enum shared between frontend and backend.

**What belongs here:**
- Zod schemas that are validated on both the client (react-hook-form) and the server (NestJS pipes or Server Actions)
- The `ErrorCode` enum referenced in `GlobalExceptionFilter` (backend) and error handlers (frontend)
- Shared constants (e.g. max file size, pagination limits) used in both apps
- Shared schema fields must stay `camelCase` so backend DTOs, Swagger, Orval, and frontend consumers all align without renaming.

**What does not belong here:**
- Schemas used only by the backend -> keep them in `apps/api/src/`
- Schemas used only by the frontend -> keep them in `apps/frontend/`

**ErrorCode enum - keep it in sync:**

```ts
// packages/schemas/src/error-codes.ts
export enum ErrorCode {
  VALIDATION_ERROR = 'VALIDATION_ERROR',
  NOT_FOUND        = 'NOT_FOUND',
  UNAUTHORIZED     = 'UNAUTHORIZED',
  FORBIDDEN        = 'FORBIDDEN',
  CONFLICT         = 'CONFLICT',
  RATE_LIMITED     = 'RATE_LIMITED',
  INTERNAL_ERROR   = 'INTERNAL_ERROR',
}
```

When adding a new error code: add it here first, then use it in both the backend filter and the frontend handler. Never hardcode error code strings in either app.

## `packages/tsconfig/`

Contains shared TypeScript configurations extended by all apps and packages.

```
packages/tsconfig/
  base.json        <- strict mode, common compiler options
  nextjs.json      <- extends base, Next.js-specific settings
  expo.json        <- extends base, React Native / Expo settings
  nestjs.json      <- extends base, NestJS / Node settings
```

**`strict: true` is set in `base.json` and must not be overridden.** If an app `tsconfig.json` sets `strict: false`, remove it.

When adding a new compiler option that applies to all projects, add it to `base.json`. When it is framework-specific, add it to the relevant file only.

## Adding a New Package

1. Create `packages/<name>/` with `package.json`, `tsconfig.json`, and `src/index.ts`.
2. Extend `packages/tsconfig/base.json` in the new `tsconfig.json`.
3. Add the package as a workspace dependency in the consuming app's `package.json`.
4. Add the build task to `turbo.json` with correct `dependsOn` so it builds before the apps that use it.
5. Export only from `src/index.ts` - do not import deep paths from consuming apps.
